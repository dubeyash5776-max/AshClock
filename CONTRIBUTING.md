# Contributing to AshClock

First of all, thank you for considering contributing to AshClock! ❤️

Whether you're reporting a bug, suggesting a new feature, improving documentation, or submitting code, your help is greatly appreciated.

## Ways to Contribute

- 🐛 Report bugs
- 💡 Suggest new features
- 🎨 Improve the UI or design
- 📖 Improve documentation
- ⚡ Optimize performance
- 🧹 Refactor code

## Before You Start

Before making significant changes, please open an Issue to discuss your idea.

This helps avoid duplicate work and ensures the proposed change aligns with the project's goals.

## Development Guidelines

- Keep the code clean and readable.
- Use meaningful comments when necessary.
- Maintain the minimalist design philosophy of AshClock.
- Test your changes before submitting.

## Pull Requests

When submitting a Pull Request:

1. Keep changes focused on a single feature or fix.
2. Update documentation if necessary.
3. Ensure the skin loads correctly in Rainmeter.
4. Write a clear description of what has changed.

## Design Philosophy

AshClock is built around four core principles:

- Minimal
- Lightweight
- Elegant
- Customizable

Please keep these principles in mind when contributing.

## Questions?

If you have any questions or ideas, feel free to open an Issue.

Thank you for helping make AshClock better! 🌙
